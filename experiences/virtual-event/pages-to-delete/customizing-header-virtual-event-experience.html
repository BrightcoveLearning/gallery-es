Personalización del encabezado de una experiencia de evento virtual

En este tema, aprenderá a personalizar el encabezado de las páginas de Virtual Event Experience.

El encabezado aparecerá en la parte superior de todas las páginas. El encabezado se construye a partir de los valores del archivo de configuración. El encabezado se compone de los siguientes componentes:

  • El logotipo de cabecera
  • Enlaces
  • Iconos sociales
  • Buscar

El logotipo de cabecera

La imagen del encabezado está especificada por el images objeto en el archivo de configuración.

  • header_logo- URL de la imagen que aparecerá en el encabezado

Se admiten archivos.png y .jpg. El logotipo del encabezado se reducirá proporcionalmente a 178 píxeles de ancho.

Enlaces

Por defecto, el HOGAR , TRANSMISIÓN EN VIVO y SESIONES BAJO DEMANDA los enlaces están codificados en la plantilla:

  • CASA - Va a inicio, parte superior de la página.
  • TRANSMISIÓN EN VIVO - Va a la colección destacada (reproductor de video superior)
  • SESIONES BAJO DEMANDA - Va a la sección de pistas

Se pueden agregar enlaces adicionales editando el header_links objeto en el archivo de configuración.

  • link_url- URL de destino
  • label- Etiqueta para aparecer en el encabezado

Iconos sociales

De forma predeterminada, el encabezado mostrará iconos sociales:

  • Facebook
  • Instagram
  • Gorjeo
  • LinkedIn
  • YouTube
Los iconos sociales que aparecen en el encabezado se pueden editar / eliminar editando el social objeto en el archivo de configuración. Para obtener información, consulte Configuración de los iconos sociales para una experiencia de evento virtual.

Buscar

Para habilitar la búsqueda y mostrar el campo de búsqueda, habilitar la búsqueda de la experiencia. De forma predeterminada, el campo de búsqueda en el encabezado mostrará una sola línea para la entrada de texto.

El cuadro de búsqueda en el ejemplo se ha diseñado para mostrar un cuadro alrededor del campo de búsqueda.

Estilo específico del encabezado

Enlaces en dispositivos móviles

De forma predeterminada, los enlaces agregados al encabezado no aparecerán al usar un dispositivo móvil. El siguiente JavaScript resuelve ese problema.


for (let i = 0, l = ciabConfig.header_links.length; i < l; i++) {
  let link = ciabConfig.header_links[i];
  let li = document.createElement('li');
  let ul = document.getElementById('main-nav-cat');
  li.className = 'main-nav-category main-nav-category-home';
  li.innerHTML = ('' + link.label + ''); ul.appendChild(li);
}

Dar formato al campo de búsqueda

De forma predeterminada, el campo de búsqueda mostrará una sola línea para la entrada de texto. El siguiente CSS da formato al campo de búsqueda como un cuadro de texto.


.bc-custom-header-nav .bc-nav-menu .main-header-search-box {
    margin-right: 45px;
    width: 170px;
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input {
	background-color: transparent;
	color: #fff;
	margin-top: -10px;
	font-family: 'Montserrat', sans-serif;
	font-size: 14px;
	font-weight: 500;
	line-height: .88;
	text-transform: uppercase;
	display: block;
    padding: 2px 5px 2px 30px;
	width: inherit;
	border: 1px solid #fff;
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input::-webkit-input-placeholder {
	text-transform: uppercase;
	color: #fff
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input::-moz-placeholder {
	text-transform: uppercase;
	color: #fff
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input:-ms-input-placeholder {
	text-transform: uppercase;
	color: #fff
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input::-ms-input-placeholder {
	text-transform: uppercase;
	color: #fff
}

.bc-custom-header-nav .bc-nav-menu .main-header-search-box input.main-header-search-input::placeholder {
	text-transform: uppercase;
	color: #fff
