    Uso del control de acceso basado en roles

    En este tema aprenderá cómo utilizar el control de acceso basado en roles para restringir los vídeos que aparecen en una experiencia basada en el rol del usuario.

    Si se requiere que los usuarios inicien sesión en su Experiencia de evento virtual, se puede utilizar el control de acceso basado en roles para filtrar los vídeos que se muestran a los usuarios en función de su rol. El control de acceso basado en roles se basa en un atributo SAML especificado en el objeto de usuario devuelto por el sistema de administración de usuarios (es decir, sistema de registro, inicio de sesión único). Las reglas se definen en función de las funciones de usuario y las etiquetas de Video Cloud. Brightcove recomienda que todos los vídeos tengan una etiqueta para identificarlos para el evento, además de etiquetas adicionales para cualquier filtrado basado en roles que tenga que producirse.

    Para configurar el control de acceso basado en roles, comience determinando cómo se restringirá el contenido y cree un archivo de configuración de acceso basado en roles adecuado. Algunas configuraciones de ejemplo se proporcionan en las siguientes secciones. A continuación, asocie el archivo de configuración basado en roles con la experiencia. Por último, etiquetar los vídeos apropiadamente.

    Nota: Al crear colecciones de vídeos, las listas de reproducción no deben utilizarse para los criterios de recopilación, ya que las listas de reproducción no admiten el filtrado de control de acceso basado en roles. Utilice criterios personalizados en su lugar.

    Creación de un archivo de configuración de acceso basado en roles

    El control de acceso basado en roles se configura mediante un archivo de configuración de acceso basado en roles independiente. El formato del archivo es el siguiente:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {}
}

    Los valores de propiedad del archivo controlan cómo se aplica el acceso a la experiencia.

    • AdvancedRoleFilter - Cuando se establece en true , habilita el control de acceso basado en roles
    • DefaultIncludeTags - Etiquetas que deben estar en todos los vídeos para que se muestren en la Experiencia de evento virtual. Esto le permite mostrar sólo un subconjunto de vídeos en la cuenta en la Experiencia de evento virtual.
    • DefaultIgnoreTags - Los vídeos con estas etiquetas no se mostrarán a los usuarios que no formen parte de un rol
    • roleAttribute - Atributo SAML en el registro de usuario que regresa del sistema de administración de usuarios. Se puede utilizar cualquier atributo SAML.
    • ValueDelimiter - Delimitador utilizado para separar varios roles (si se pueden devolver varios roles)
    • IgnoreByRole - Aumenta el defaultIgnoreTags ignorando adicionalmente más etiquetas basadas en el rol del usuario. Esto se utiliza para eliminar aún más vídeos que el usuario puede ver en función de su rol.
    • IncludeMyRole - Los usuarios con rol asignado solo verán vídeos con la etiqueta especificada
    • AllowByRole - Aumenta el defaultIncludeTags añadiendo adicionalmente más etiquetas basadas en el rol del usuario. Esto se utiliza para mostrar más vídeos a un usuario en función de su rol. Los usuarios con el rol especificado verán los vídeos etiquetados con, así defaultIncludeTags como las etiquetas listadas aquí.
    En las secciones siguientes se proporcionan algunos archivos de configuración de ejemplo.

    Función de prima única

    En este escenario, los usuarios generales pueden ver la mayor parte del contenido, pero un subconjunto de contenido etiquetado con la etiqueta «premium» solo está disponible para los usuarios con el rol de «premium_user». Los usuarios Premium aún pueden ver todo el contenido que los usuarios generales pueden ver. El uso de una etiqueta «virtual_event» restringe el sitio para mostrar solo el contenido de la cuenta etiquetada específicamente para el evento:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "premium_user": ["premium"]
     }
}

    Acceso al contenido en niveles

    Similar al primer escenario, pero con varios niveles donde cada nivel obtiene acceso a más contenido y aún puede ver todo el contenido de los niveles inferiores. Las funciones de usuario son «tier_1», «tier_2» y «tier_3», mientras que los usuarios que no tengan uno de estos roles no verán ningún contenido en niveles. Las etiquetas son «tier1», «tier2» y «tier3» respectivamente:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tier1", "tier2", "tier3"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "tier_1": ["tier1"],
          "tier_2": ["tier1", "tier2"],
          "tier_3": ["tier1", "tier2", "tier3"]
     } 
}

    Dividir acceso al contenido

    En este ejemplo, los dos roles de usuario solo ven vídeos etiquetados específicamente para su rol. Un usuario sin un rol no verá el contenido de ninguna de las etiquetas:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tag1", "tag2"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "role1": ["tag1"],
          "role2": ["tag2"]
     },
     "allowByRole": {}
}

    Segmentación por idioma

    Utilizando el enfoque basado en divisiones, el contenido se puede servir en función de un atributo SAML que contenga el idioma del usuario:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": [],
     "roleAttribute": "language",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "en": ["english"],
          "es": ["spanish"],
          "jp": ["japanese"]
     },
     "allowByRole": {}
}

    Ejemplo de reglas combinadas

    Este ejemplo artificioso ilustra cómo se pueden combinar las diferentes opciones. Un usuario general verá todo el contenido etiquetado «virtual_event» que no esté etiquetado como «premium». Un usuario con la función «premium_user» verá todo el contenido general más el contenido premium, pero no verá el contenido etiquetado con «promocional». Un usuario con un rol «trial_user» verá contenido general y contenido premium, pero también verá el contenido etiquetado con «promocional». Un usuario con el rol «expired_user» sólo verá vídeos etiquetados como promocionales:

    
{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {
          "premium_user": ["promotional"]
     },
     "includeByRole": { "expired_user": ["promotional"]
     },
     "allowByRole": {
          "trial_user": ["promotional"],
          "premium_user": ["premium"]
     }
}

    Asignación de un perfil de acceso basado en roles

    Una vez que haya creado un perfil de acceso basado en roles para controlar el acceso al contenido, asócielo a la experiencia.

    1. Copie el perfil de control de acceso en el portapapeles.
    2. Abra el módulo de Gallery.
    3. Edite la experiencia.
    4. Haga clic en APARIENCIA Y COMPORTAMIENTO > Personalizado en la navegación izquierda.
    5. Pegue la configuración del perfil de acceso en el campo JSON de plantilla personalizada .
      Nota: Si no se muestra este campo, la cuenta no se ha habilitado para el control de acceso basado en roles.
    6. Haga clic en Guardar.

    Asegúrese de que vuelve a publicar la experiencaso de que aparezcan los cambios.  

    Exposición de atributos SAML

    Para que la experiencia pueda acceder al atributo de rol desde el proveedor de sistema de registro o SSO/registro, asegúrese de que la configuración de SSO esté configurada correctamente.

    1. Abra el módulo de Gallery.
    2. Haga clic en el Configuraciones enlace en la página de inicio de la Galería.
    3. En el panel de navegación de la izquierda, haga clic en Perfiles de control de acceso. Se mostrará una lista de perfiles de control de acceso creados.
    4. Haga clic en el icono de edición del perfil que está utilizando Experiencia de evento virtual.
    5. Confirme que la opción Exponer todos los atributos de la parte inferior de la página está marcada.
    6. Haga clic en Guardar.
