Configuración de dominios personalizados y SSL

{{ page.description }}
 

Cuando crea una experiencia de portal por primera vez, se le asigna un dominio aleatorio con el formato de <site-slug>.bcvp0rtal.com . Por ejemplo, http://site-148393.bcvp0rtal.com . Cuando se publique el sitio, los espectadores podrán acceder al sitio en este dominio. Gallery ofrece la posibilidad de publicar sitios de vídeos en un dominio proporcionado por Brightcove o en su propio dominio personalizado.

Gallery Portal Experiences también se puede asignar a varias URL, lo que permite a los editores utilizar el mismo sitio para diferentes funciones sin tener que duplicar el sitio. Por ejemplo, si tiene un sitio de videos que desea usar en una campaña y como su portal de videos principal, puede marcar diferentes URL y asignarlas todas al mismo sitio. No es necesaria la duplicación del sitio y puede eliminar fácilmente las URL cuando ya no las necesite.

Nota: Cuando trabaje con certificados SSL, en ningún caso comparta su clave privada ni la envíe a Brightcove Support.

Uso de un dominio proporcionado por Brightcove

Los dominios proporcionados por Brightcove le permiten crear rápida y fácilmente un dominio para su sitio sin tener que trabajar con su departamento de TI para crear un CNAME personalizado o registrar un nuevo nombre de dominio con un registrador. Están disponibles "listos para usar" y se pueden implementar con un clic de un botón. Hay tres dominios proporcionados por Brightcove:

  • brightcovegallery.com
  • gallery.video
  • bcvp0rtal.com (que está asignado a los sitios de forma predeterminada)

Todos ellos admiten SSL. Para acceder al sitio SSL, simplemente agregue https a la URL. Por ejemplo, https://marquee.brightcovegallery.com

Usando un dominio personalizado (editor registrado)

Los sitios de videos también pueden alojarse en su propio dominio personalizado que registre. Esto le permite marcar la URL y mantenerla coherente con la construcción de nomenclatura de su sitio web / URL actual. Esto requiere que registre el nombre de dominio y luego configure un registro CNAME personalizado con su proveedor de alojamiento. Ver el sección a continuación para obtener información sobre cómo configurar un registro CNAME.

Creando un dominio personalizado

Para crear un dominio personalizado, siga estos pasos:

  1. Abra el módulo Galería.
  2. Haga clic en el Ajustes enlace en la página de inicio de la Galería.
  3. En la barra de navegación de la izquierda, haga clic en el DOMINIOS PERSONALIZADOS Y SSL Enlace.
  4. Hacer clic Crear dominio personalizado.
  5. Seleccione una opción de dominio para usar:
    • brightcovegallery.com - El dominio del sitio tendrá el formato <your domain>.brightcovegallery.com.
    • gallery.video - El dominio del sitio tendrá el formato <your domain>.gallery.video .
    • Personalizado - Seleccione alojar el sitio en su propio dominio personalizado. Al seleccionar esta opción, también deberá configurar un registro CNAME con su proveedor de alojamiento.
  6. Ingrese el nombre de dominio. Se mostrará un mensaje para indicar si el dominio es válido o inválido. Si el nombre de dominio no es válido, ingrese un nuevo nombre de dominio.
  7. Si está publicando el sitio utilizando un dominio proporcionado por Brightcove, hay una opción para Aplicar SSL en este dominio. Cheque Aplicar SSL en este dominio (recomendado) para habilitar un redireccionamiento desde la versión no SSL del sitio si un espectador accede a él.
  8. Si publica el sitio en su propio dominio personalizado, tiene la opción de proteger el sitio mediante SSL. Comprobar el Utilice SSL casilla de verificación para proteger el sitio mediante SSL. Para obtener información sobre el uso de SSL, consulte Asegurar una experiencia de portal con SSL.
  9. Cheque Habilitar redireccionamiento para redirigir esta URL a otra URL. Por lo general, esto solo se verificará después de que este dominio se haya utilizado durante un tiempo. Cuando se selecciona esta opción, ingrese el Redireccionar URL esta URL existente debe ser redirigida. Cheque Utilice una redirección permanente (código de estado HTTP 301) para indicar que la redirección es permanente. Cuando una araña de motor de búsqueda encuentra un código de estado 301 en el encabezado de respuesta de una página web, entiende que esta página web ya no existe y reemplaza la URL indexada con la nueva. Esta opción conservará el tráfico y la clasificación de la página web anterior.
  10. Hacer clic Ahorrar. Se mostrará una lista de dominios configurados.
    Nota: Cuando utilice un dominio proporcionado por Brightcove, el estado de SSL siempre será Previsto. Esto indica que al agregar https:// al principio de la URL del sitio, la transmisión de datos entre su navegador y el sitio de la Galería estará segura.
  11. Cree dominios adicionales según sea necesario.

Para editar o eliminar un dominio, haga clic en editar () o eliminar () botones.

Una vez que se han creado los dominios personalizados, se pueden asignar a un sitio como parte de las propiedades del sitio. Para obtener información, consulte Asignar un dominio personalizado a una experiencia de portal.

Configuración de registros CNAME con su proveedor de alojamiento

Nota: Los registros CNAME solo deben configurarse cuando se asignarán dominios personalizados a un sitio. NO es necesario crear registros CNAME para sitios que utilicen dominios proporcionados por Brightcove.

Los pasos para configurar un registro CNAME (nombre canónico) variarán según su proveedor de alojamiento. Debe configurar un registro CNAME para cada dominio personalizado que utilizará. Si su sitio está configurado para usar SSL, el registro CNAME debe apuntar a la URL de CloudFront. Ver Asegurar una experiencia de portal con SSL para información. Tenga en cuenta que los dominios personalizados deben incluir un subdominio para poder configurarse como CNAME, por ejemplo: video debe colocarse antes brightcovelearning.com para que pueda configurar mi CNAME.

Los valores CNAME se muestran como parte de la configuración de SSL y dominios personalizados.

En este ejemplo, para el dominio personalizado de video.brightcovelearning.com , Agregaría un CNAME grabar y agregar una gran cantidad de video y apúntalo a d3mkhzp6k6x3hl.cloudfront.net.

Tenga en cuenta que navegar directamente a una URL de Cloudfront devolverá un error de "página no encontrada" (404). A curl El comando se puede utilizar para verificar la configuración del dominio. Para este ejemplo:

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

La declaración curl debe devolver el contenido de la página de inicio del sitio.

Es posible que deba comunicarse con su administrador de red o proveedor de alojamiento web para obtener ayuda con este paso. Recuerde que los cambios de DNS pueden tardar hasta 48 horas en surtir efecto.

Protección de sitios con SSL

Hay dos opciones para asegurar una experiencia de portal con SSL:

  1. Para los sitios que se publican utilizando el slug de sitio asignado (es decir, <site-slug>.bcvp0rtal.com ) o que utilizan un dominio suministrado por Brightcove, no es necesaria ninguna configuración para admitir https. Añadiendo https: // al principio de la URL asegurará la transmisión de datos entre su navegador y el sitio de la Galería. También puede seleccionar Aplicar SSL en este dominio (recomendado) para habilitar un redireccionamiento desde la versión no SSL del sitio si un espectador accede a él.
  2. Para los sitios que se alojan con su propio dominio personalizado, marque la casilla Utilice SSL opción al ingresar su nombre de dominio. Luego se le pedirá que ingrese la información de su certificado. Para más información, ver Asegurar una experiencia de portal con SSL.
